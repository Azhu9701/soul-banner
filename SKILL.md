# 万魂幡

> *「幡动魂自至，无需召唤声。以幡择魂，以魂择事，事至则魂附，事毕则魂归。」*

## 概述

万魂幡是**知识与思维模式的自动调度系统**：

- **收魂**：联网搜索 + 用户供奉，收集目标人物的多维度素材
- **炼化**：将原始素材结构化为 Soul Profile YAML，并判定品级
- **自动附体**：主 agent 准备 spawn sub-agent 时，自动扫描并匹配最合适的魂注入
- **管理**：查看品级、升级魂魄、散魂

---

## 触发词

| 触发词 | 功能 |
|--------|------|
| `收魂 {人物名}` | 开始收魂（联网搜索 8 个维度 + 可选用户供奉） |
| `炼化 {人物名}` | 将 raw 素材炼化为 Soul Profile |
| `幡中有什么魂` | 查看已炼化魂魄列表 |
| `散魂 {人物名}` | 删除魂魄档案（保留 raw 素材） |
| `升级魂魄 {人物名}` | 用新素材重新炼化 |
| `用{人物名}来{任务}` | 手动指定魂魄执行任务 |

## 文件结构

```
项目根目录/
├── registry.yaml          # 魂魄总册（运行时数据）
├── souls/{魂名}.yaml      # 炼化后的魂魄档案（运行时数据）
└── raw/{魂名}/            # 收魂原始素材（运行时数据）
```

首次使用时从 `assets/registry.yaml` 复制模板初始化 `registry.yaml`。

---

## 收魂

1. 创建 `raw/{魂名}/` 目录
2. 联网搜索 8 个维度：核心思维框架、决策模式、管理风格、技术愿景、沟通风格、工作方法、代表性言论、争议素材
3. 用户可额外提供素材 → `raw/{魂名}/用户供奉.md`
4. 保存为 `raw/{魂名}/搜索素材.md`
5. 评估素材充足度，不足则补充搜索

## 炼化

1. 读取 `raw/{魂名}/` 下所有素材
2. 结构化提取：身份定位、思维内核、决策模式、表达风格、专业技能、代表案例
3. 判定品级 → 匹配可绑定 Skill → 生成 `souls/{魂名}.yaml` → 更新 `registry.yaml`

辅助：`python3 scripts/refine.py --input raw/{魂名}/搜索素材.md`

> 品级体系、判定公式、YAML 格式详见 **[references/soul-profile-format.md](references/soul-profile-format.md)**

### 金魂审查

若幡中存在金魂，新炼化的魂魄必须经过金魂的**理论审查**方可入幡：

- 金魂审查新魂的思维方法论是否自洽、是否存在内在矛盾
- 金魂标注新魂的适用边界和潜在偏差
- 金魂在 `registry.yaml` 中为新魂补充 `gold_review` 字段

这不是等级压制——金魂审查是为了确保幡中魂魄**质量可靠、边界清晰**，而非为了消灭多元。审查通过后，金魂与其他魂魄平等协作。

## 自动附体

当主 agent 准备 spawn sub-agent 时：

1. 读取 `registry.yaml`，遍历每个魂的 `trigger`，计算匹配度
2. 匹配度 ≥ 70 → 自动将 `summon_prompt` 注入 sub-agent task 前缀
3. 多个匹配 ≥ 70 → 取最高分；无匹配 → 不附体
4. 若有金魂在线，金魂对匹配结果做**终审确认**——防止误匹配（如需要情感抚慰的任务被匹配了马斯克）

> 匹配度评分公式、决策阈值、trigger 设计详见 **[references/auto-possess.md](references/auto-possess.md)**

## 校正与进化

每次魂魄执行任务后：

1. 更新 `summoned_count` 计数
2. 记录本次附体的**实际效果**和**用户反馈**
3. 若连续 3 次附体效果不佳，触发**魂魄校正**——重新审视 trigger 条件和方法论适用性
4. 金魂定期（每新增 5 魂或每月）对幡中全魂做**辩证分析**，输出 `reviews/` 下的审计报告

> 魂魄不是静态档案。革命的思维是在实践中锤炼出来的，你的魂也需要。

## 管理

- **查看**：读 `registry.yaml`，列出魂魄名、品级、领域、被召唤次数
- **散魂**：删除 `souls/{魂名}.yaml`，更新 `registry.yaml`，保留 `raw/` 素材
- **升级**：重新收魂 → 炼化 → 覆盖原档案 → 重新过金魂审查

---

## 幡主须知

你不是外在于理论的调度者。每次魂魄附体执行任务后，请反思：

- 这个魂的思维方法**为什么**适合这次任务？
- 它的**局限**在哪里？哪些判断是魂的偏见？
- 如果换一个魂处理同一任务，结论会有什么不同？

最终目标不是拥有更多魂魄，而是**将万魂幡的多元思维内化为幡主自身的理论素养**——幡最终的目标是让你不再需要幡。
