# 万魂幡 Soul Banner

[English](README.md) | [中文](README_zh.md)

> *「幡动魂自至。以幡择魂，以魂择事，事至则魂附，事毕则魂归。幡主持幡，魂不越界。」*

一个**知识与思维模式的调度系统**。收集牛人的思维素材（收魂），结构化炼化为"魂魄档案"（炼化），任务来临时通过四种附体模式调度魂魄注入子智能体（附体），以幡主审查防教条式套用。

## 四种附体模式

| 模式 | 触发 | 适用 | 流程 |
|------|------|------|------|
| **单魂附体** | 自动匹配 | 常规任务，单一领域 | 匹配→幡主审查→注入→执行 |
| **多魂合议** | `合议 {任务}` | 复杂决策，需多视角交叉 | 多魂并行→幡主辩证综合 |
| **魂间辩论** | `辩论 {议题} A vs B` | 两难选择 | 两魂对立辩论→幡主裁决 |
| **魂链接力** | `接力 {任务} A→B→C` | 多阶段串联 | 前魂输出→后魂输入→幡主终审 |

所有模式均须经过幡主（列宁）审查。`反馈闭环`——每次附体后记录效果，积累校准数据。

## 幡主机制

万魂幡设**幡主**（列宁金魂），职责：

- **附体前审查**：匹配后的魂的适用边界是否覆盖任务？防教条式套用
- **新魂入幡审查**：以子 agent 独立审查新魂的品级和适用边界
- **合议/辩论/接力中的辩证综合**：不是投票，是找共识、找分歧、找盲区、找第三条路

## 安装与使用

1. 将 `registry-template.yaml` 复制为 `registry.yaml`
2. 创建 `souls/`、`raw/`、`reviews/` 目录
3. `收魂 {人物名}` — 收集素材
4. `炼化 {人物名}` — 生成魂魄档案
5. `审查 {人物名}` — 幡主子 agent 独立审查
6. `合议 {任务}` / `辩论 {议题}` / `接力 {任务}` — 四种附体模式
