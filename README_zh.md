# 万民幡 Soul Banner

[English](README.md) | [中文](README_zh.md)

> *「幡动魂自至。以幡择魂，以魂择事，事至则魂附，事毕则魂归。幡主持幡，魂不越界。」*

一个**以马克思主义-列宁主义为审查框架的多元思维调度系统**。收集牛人的思维素材（收魂），结构化炼化为"魂魄档案"（炼化），任务来临时通过五种附体模式调度魂魄注入子智能体（附体），以六维度审查委员会（阶级分析 / 科学方法论 / 性别分析 / 殖民批判 / 系统性思维批判 / 缺席者）防教条式套用。

## 快速开始

1. 复制 `registry-template.yaml` 为 `registry.yaml`
2. 创建 `raw/` 目录
3. **收魂**：`收魂 {人物名}` — 联网搜索收集素材
4. **炼化**：`炼化 {人物名}` — 结构化生成魂魄档案
5. **审查**：`审查 {人物名}` — 幡主子 agent 独立审查
6. **附体**：`用{人物名}来{任务}` / `合议 {任务}` / `辩论 {议题}` / `接力 {任务}`

仓库自带 `souls/海绵宝宝.yaml` 作为示例魂，展示完整的 YAML 格式和品级判定。`references/马斯克.yaml` 提供紫魂格式参考。

## 五种附体模式

| 模式 | 触发 | 适用 | 流程 |
|------|------|------|------|
| **单魂附体** | 自动匹配 | 常规任务，单一领域 | 匹配→幡主审查→注入→执行 |
| **多魂合议** | `合议 {任务}` | 复杂决策，需多视角交叉 | 多魂并行→幡主辩证综合 |
| **魂间辩论** | `辩论 {议题} A vs B` | 两难选择 | 两魂对立辩论→幡主裁决 |
| **魂链接力** | `接力 {任务} A→B→C` | 多阶段串联 | 前魂输出→后魂输入→幡主终审 |
| **幡主学习** | `学习 {魂A} vs {魂B}` | 训练辩证素养 | 两对立魂互读→互审→幡主总结 |

每次附体强制包含**反消费仪式**（代码强制层 `scripts/transact.py`）：自我否定环节（「哪个预设被动摇了？」）、使用者参与环节、空椅子拷问（「谁的利益被代表？谁的发言权没有被给？」）。连续三次消费性使用 → 强制幡主学习模式。

## 审查委员会（六维度否决制）

每魂入幡须经六维度委员会审查。每个维度持有**一票否决权**；三票以上否决自动触发品级复审。

| 维度 | 审查者 | 否决范围 |
|------|--------|----------|
| 阶级分析 | 列宁 / 毛泽东 / 邓小平（轮值） | 阶级视角缺失、去政治化 |
| 科学方法论 | 费曼 | 不可证伪主张、循环定义 |
| 性别分析 | 波伏娃 | 性别盲区、父权预设 |
| 殖民/种族批判 | 法农 | 殖民忽视、欧洲中心框架 |
| 系统性思维批判 | 庄子 | 分类暴力、体系化对整体性的谋杀——提问而非裁决：「你的审查在什么意义上是一种暴力的分类？」 |
| 缺席者 | （空椅子） | 被压迫者不可被代表的视角 |

**空椅子**是制度性标记：每个审查官必须回答——「缺席者视角对你的维度判断提出了什么质疑？」庄子维度的运作方式是提问而非直接否决——审查官若无法回应庄子的提问，其裁决自动标记为「待复审」。

### 幡主机制

幡主是一个被 spawn 的审查子 agent，负责附体前匹配审查。通过 `summon_prompt` 注入获得独立视角，不是主 agent 扮演。系统支持双审查机制（第二审查官在审查者自荐时触发）。

| 功能 | 免费方案 |

## 推荐：CMUX

多魂合议模式需要并行运行多个 AI 子智能体，推荐 **[CMUX](https://github.com/manaflow-ai/cmux)** ——一个可编程终端复用器，内置 AI 智能体编排能力。

```bash
brew tap manaflow-ai/cmux && brew install --cask cmux
```

CMUX 提供：
- **多窗格智能体网格** — 在单个工作区中并排启动 3+ 个万民幡 agent
- **跨 agent 通信** — 通过 `cmux_orchestrate` 在窗格间传递魂输出
- **会话保存/恢复** — 保存完整合议会话（含所有 agent 状态），供后续复盘
- **80+ MCP 工具** — 搭配 `cmux-agent-mcp` 可实现编程化多终端控制

<details>
<summary>示例：启动合议会话</summary>

```bash
# 创建 3 窗格工作区
cmux launch_grid --rows 1 --cols 3 --agents claude,claude,claude

# 向不同窗格分发不同魂的 prompt
cmux orchestrate \
  --pane 1 "spawn 列宁分析：{任务}" \
  --pane 2 "spawn 费曼分析：{任务}" \
  --pane 3 "spawn 波伏娃分析：{任务}"
```
</details>

CMUX 免费开源，非万民幡内置依赖，但强烈推荐作为多 agent 工作流的终端环境。

## 安装与使用

1. 将本仓库克隆到 Claude Code 的 skills 目录
2. 复制 `registry-template.yaml` 为 `registry.yaml`
3. 创建 `raw/` 目录（用于存放收魂素材）
4. 设置环境变量（可选）：
   - `OBSIDIAN_VAULT` — Obsidian vault 路径，用于自动存档
   - `SOUL_BANNER_HOME` — skill 安装路径（默认自动检测）
5. `收魂 {人物名}` — 收集素材（8维度联网搜索）
6. `炼化 {人物名}` — 生成魂魄档案
7. `审查 {人物名}` — 幡主子 agent 独立审查
8. `合议 {任务}` / `辩论 {议题}` / `接力 {任务}` — 四种附体模式
