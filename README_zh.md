# 万魂幡 Soul Banner

[English](README.md) | [中文](README_zh.md)

> *「幡动魂自至。以幡择魂，以魂择事，事至则魂附，事毕则魂归。幡主持幡，魂不越界。」*

## 品级体系（六阶）

| 品级 | 符号 | 标准 | 范例 |
|------|------|------|------|
| 金魂 | 🟡 | 独立世界观，有方向判断力，体系内有自我批判机制 | 列宁、毛泽东、邓小平、费曼(有限) |
| 银魂 | 🥈 | 领域顶级实践者，方法论完整可操作，缺独立世界观 | Karpathy |
| 紫魂 | 🟣 | 资料丰富，有明确方法论，但依赖个人天赋/不可复制 | 马斯克、罗永浩、乔布斯 |
| 蓝魂 | 🔵 | 资料充足，有完整 Skill 链，思维模式独特 | 海绵宝宝 |
| 绿魂 | 🟢 | 资料中等，1-2 个可匹配 Skill | — |
| 白魂 | ⚪ | 资料极少，仅通过公开信息模拟 | — |

**金魂严格标准**：必须同时满足 (1)完整的执行方法 (2)独立的判断方法/世界观 (3)体系内的自我批判机制。缺(2)有(1) → 银魂。有(2)但方法论不可复制 → 紫魂。

## 幡主机制

万魂幡设**幡主**（默认列宁金魂），负责：

- **附体前审查**：匹配度计算后、注入 summon_prompt 前，幡主判断魂的适用边界是否覆盖当前任务
- **新魂入幡审查**：以子 agent 独立审查新炼化魂魄的品级和适用边界
- **防教条**：否决不恰当的匹配，防止机械套用

## 安装与使用

1. 将 `registry-template.yaml` 复制到项目根目录并重命名为 `registry.yaml`
2. 创建 `souls/`、`raw/`、`reviews/` 目录
3. 说「收魂 {人物名}」开始收集素材
4. 说「炼化 {人物名}」生成魂魄档案
5. 说「审查 {人物名}」让幡主子 agent 独立审查
6. 任务来临时，万魂幡匹配魂魄 → 幡主审查 → 附体
