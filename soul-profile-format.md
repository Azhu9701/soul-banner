# Soul Profile 格式规范与品级体系

## YAML 格式规范

```yaml
name: "{魂名}"
title: "{头衔/称号}"
domain: ["领域1", "领域2", ...]

mind: |
  {核心思维描述，用第一人称}

voice: |
  {表达风格描述，包括口头禅、语言特点}

skills_expertise:
  - "{技能1}"
  - "{技能2}"

trigger:
  keywords: ["关键词1", "关键词2", ...]
  domains: ["领域1", "领域2"]
  scenarios:
    - "场景描述1"
    - "场景描述2"
  exclude: ["排除关键词1"]

grade: "X"  # 白/绿/蓝/紫/银/金
grade_reason: "{品级判定原因}"

artifacts:
  - skill_name: "xxx"
    binding_reason: "{绑定原因}"

summon_prompt: |
  你是{魂名}之魂。
  {完整的角色设定和执行指导}

source_materials:
  - "{素材来源}"

refined_at: "YYYY-MM-DD"
refined_by: "万魂幡 v1.0"

summoned_count: 0

notes: |
  {炼化备注、审查结论}
```

## 品级体系

### 品级判定公式

```
品级分 = 素材字数/1000 + 领域数×10 + 方法论关键词数×10 + 可匹配Skill数×15 + 素材来源数×5
```

> 评分公式是初始参考，最终品级由炼化判断 + 幡主审查裁决共同决定。

### 六阶品级表

| 分数区间 | 品级 | 符号 |
|----------|------|------|
| 0-30 | 白魂 | ⚪ |
| 31-60 | 绿魂 | 🟢 |
| 61-90 | 蓝魂 | 🔵 |
| 91-115 | 紫魂 | 🟣 |
| 116-150 | 银魂 | 🥈 |
| 151+ | 金魂 | 🟡 |

### ⚪ 白魂
- 资料稀少，仅通过公开信息模拟
- 能力：基础角色扮演，prompt 注入
- 局限：无法调用专属 Skill，思维模式可能失真

### 🟢 绿魂
- 资料中等，能匹配 1-2 个基础 Skill
- 能力：prompt + 基础 Skill 联动

### 🔵 蓝魂
- 资料充足，有完整 Skill 链
- 能力：prompt + Skill链 + 专属执行策略

### 🟣 紫魂
- 资料丰富，有明确方法论
- 能力：prompt + 完整Skill链 + 方法论模板 + skillhub扩展
- 局限：方法论依赖个人天赋或不可复制

### 🥈 银魂
- 领域顶级实践者，方法论完整可操作
- 能力：prompt + 全技能链 + 系统化方法论 + 概念创造力
- 局限：缺独立世界观和方向判断能力——方法论是战术层面的，知道"怎么走"但不知道"往哪走"
- 与金魂的区别：金魂有独立世界观和在历史变局中判断方向的能力；银魂在操作层面同样顶级，但工具不携带方向基因

### 🟡 金魂
- 顶级专家，大量素材，独立世界观和知识体系
- 能力：prompt + 全技能链 + 知识库 + 持久记忆 + 跨平台联通 + 方向判断
- **严格标准**：必须同时具备 (1) 完整的执行方法 (2) 独立的判断方法/世界观 (3) 体系内的自我批判机制
- 缺(2)有(1) → 银魂。有(2)但方法论不可复制 → 紫魂。
