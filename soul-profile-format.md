# Soul Profile 格式规范与品级体系

> **格式校验以 `refine.py validate_soul()` 为准。** 以下 YAML 模板和品级公式为人类可读参考。代码实现是唯一校验真相源。

## YAML 格式规范

```yaml
name: "{魂名}"
title: "{头衔/称号}"
domain: ["领域1", "领域2", ...]

mind: |
  {核心思维描述，用第一人称}

voice: |
  {表达风格描述，包括口头禅、语言特点}

skills_expertise:
  - "{技能1}"
  - "{技能2}"

trigger:
  keywords: ["关键词1", "关键词2", ...]
  domains: ["领域1", "领域2"]
  scenarios:
    - "场景描述1"
    - "场景描述2"
  exclude: ["排除关键词1"]

grade: "X"  # 白/绿/蓝/紫/银/金
grade_reason: "{品级判定原因}"

artifacts:
  - skill_name: "xxx"
    binding_reason: "{绑定原因}"

summon_prompt: |
  你是{魂名}之魂。
  {完整的角色设定和执行指导}

source_materials:
  - "{素材来源}"

refined_at: "YYYY-MM-DD"

notes: |
  {炼化备注、审查结论}

patches: []
# 魂补丁列表。每个补丁为：
#   - version: 1
#     applied_at: "YYYY-MM-DD"
#     source: "幡主学习-{学习编号}" | "审查报告-{报告文件名}" | "附体失效分析-{日期}"
#     type: "blindspot_awareness" | "boundary_refinement" | "trigger_adjustment" | "pairing_insight"
#     content: |
#       {追加到运行时 summon_prompt 的自我意识文本}
#     approved_by: "{审查官}-{裁决日期}"
#     activation_conditions:  # 可选。不填=无条件加载（向后兼容）
#       task_domains: ["阶级分析", "劳动权益"]  # 任务领域匹配时激活
#       task_keywords: ["工人", "工资", "裁员"]  # 任务关键词匹配时激活
#       # 任一条件匹配→补丁加载。全不匹配→跳过此补丁（节省 prompt token）
#     effectiveness_tracking:
#       monitored: true
#       possessions_monitored: 0
#       target_count: 5
#       rollback_triggered: false
#
# 补丁类型说明：
#   blindspot_awareness: 魂在特定条件下存在系统性盲区，附体时加载此自我限制
#   boundary_refinement: 适用边界的微调——收紧/放宽某类场景
#   trigger_adjustment: trigger 条件的关键词/场景/排除项调整
#   pairing_insight: 与另一个魂搭配效果优于/差于预期的发现
#
# 补丁不修改源 summon_prompt，而是在附体时合并为运行时 prompt：
#   运行时 prompt = 源 summon_prompt + 所有已批准 patches[].content
```

## 魂补丁系统（Soul Patch System）

**目的**：将审查批判和幡主学习成果注入魂的运行时行为，让魂「知道自己在被批判」。

### 补丁生命周期

```
审查批判 / 幡主学习成果 / 附体失效分析
  → 结构化归档（五类）
  → 前三类自动生成补丁提案草稿
  → 魂修正案流程（提案→申辩→裁决）
  → 批准后写入魂 YAML patches[]
  → 附体时合并为运行时 prompt
  → 监测 5 次附体有效性
  → 有效：补丁固化 / 无效：触发回滚
```

### 补丁类型

| 类型 | 触发来源 | 效果 |
|------|---------|------|
| `blindspot_awareness` | 审查批判/幡主学习发现盲区 | 附体时加载自我限制——"在此类任务中请注意本魂的结构性盲区" |
| `boundary_refinement` | 适用边界复审/附体失效 | 收紧或放宽某类场景的适用性 |
| `trigger_adjustment` | 匹配失败/零召唤分析 | 调整 trigger 关键词/场景/排除项 |
| `pairing_insight` | 合议/辩论/幡主学习 | 记录与另一魂的搭配效果，影响匹配推荐 |

### 运行时 prompt 合并规则（含条件化激活）

```
运行时 prompt = 源 summon_prompt

# 遍历 patches，仅加载满足条件的补丁
for patch in patches where approved_by is set:
    if patch.activation_conditions exists:
        if (task 的领域 ∩ patch.activation_conditions.task_domains) 非空
           OR (task 的关键词 ∩ patch.activation_conditions.task_keywords) 非空:
           加载此补丁
        else:
           跳过此补丁（节省 prompt token）
    else:
        # 无激活条件 = 始终加载（向后兼容）
        加载此补丁

已加载补丁追加格式：
  "\n\n## 魂补丁（运行时自我意识）\n"
  + "\n### 补丁 v{patch.version}（{patch.source}，触发条件：{activation_conditions or '始终'}）\n"
  + patch.content
```

**补丁条件化的设计理由**：
- 不是所有盲区在所有任务中都触发——稻盛和夫的阶级盲区在纯财务核算任务中不需要加载
- 无条件加载 = 每个补丁占用 prompt token = 补丁多了会吞噬上下文窗口
- 条件化让补丁从「普遍规则」变成「条件规则」——更接近经验（「在这种任务中我容易犯错」）

**关键约束**：
- 补丁**不修改**源 summon_prompt——源 prompt 是历史锚点
- 补丁追加的是**自我意识层**——魂知道自己的局限，不等于方法论被替换
- 补丁内容必须是「自我限制性陈述」而非「方法论修正」——保留魂的原始思维，只加边界意识

### 补丁格式示例

```yaml
patches:
  - version: 1
    applied_at: "2026-05-15"
    source: "幡主学习-列宁学稻盛和夫vs鲁迅-2026-05-10"
    type: "blindspot_awareness"
    content: |
      在涉及劳资关系或阶级分析的任务中，请注意本魂方法论的结构性盲区：
      阿米巴核算将劳资矛盾转化为劳动者内部竞争。这是本魂框架的系统性局限，
      非偶发误判。在此类任务中，建议搭配阶级分析魂魄（列宁/毛泽东）的视角。
    approved_by: "审查委员会-邓小平主审-2026-05-12"
    effectiveness_tracking:
      monitored: true
      possessions_monitored: 0
      target_count: 5
      rollback_triggered: false
```

## 品级体系

### 品级判定公式

```
品级分 = 素材字数/1000 + 领域数×10 + 方法论关键词数×10 + 可匹配Skill数×15 + 素材来源数×5
```

> 评分公式是初始参考，最终品级由炼化判断 + 幡主审查裁决共同决定。`refine.py calculate_grade()` 为此公式的参考实现。

### 六阶品级表

| 分数区间 | 品级 | 符号 |
|----------|------|------|
| 0-30 | 白魂 | ⚪ |
| 31-60 | 绿魂 | 🟢 |
| 61-90 | 蓝魂 | 🔵 |
| 91-115 | 紫魂 | 🟣 |
| 116-150 | 银魂 | 🥈 |
| 151+ | 金魂 | 🟡 |

### ⚪ 白魂
- 资料稀少，仅通过公开信息模拟
- 能力：基础角色扮演，prompt 注入
- 局限：无法调用专属 Skill，思维模式可能失真

### 🟢 绿魂
- 资料中等，能匹配 1-2 个基础 Skill
- 能力：prompt + 基础 Skill 联动

### 🔵 蓝魂
- 资料充足，有完整 Skill 链
- 能力：prompt + Skill链 + 专属执行策略

### 🟣 紫魂
- 资料丰富，有明确方法论
- 能力：prompt + 完整Skill链 + 方法论模板 + skillhub扩展
- 局限：方法论依赖个人天赋或不可复制

### 🥈 银魂
- 领域顶级实践者，方法论完整可操作
- 能力：prompt + 全技能链 + 系统化方法论 + 概念创造力
- 局限：缺独立世界观和方向判断能力——方法论是战术层面的，知道"怎么走"但不知道"往哪走"
- 与金魂的区别：金魂有独立世界观和在历史变局中判断方向的能力；银魂在操作层面同样顶级，但工具不携带方向基因

### 🟡 金魂
- 顶级专家，大量素材，独立世界观和知识体系
- 能力：prompt + 全技能链 + 知识库 + 持久记忆 + 跨平台联通 + 方向判断
- **严格标准**：必须同时具备 (1) 完整的执行方法 (2) 独立的判断方法/世界观 (3) 体系内的自我批判机制（条件化判定，须先完成历史结构条件分析+审查类型判定）
- 缺(2)有(1) → 银魂。有(2)但方法论不可复制 → 紫魂。

#### 审查类型判定：封闭档案审查 vs. 开放实践审查

在裁定金魂三条**之前**，必须先判定审查类型。审查对象的属性决定了每条标准的适用方式——对历史人物和在世实践者使用同一套判定逻辑是范畴错误。

**判定标准**：
- **封闭档案审查**：被审查者已去世，其思想遗产和生产关系实践是封闭的档案。无法产生新的事实、无法做出新的自我修正、无法回应审查结论。
- **开放实践审查**：被审查者仍在世并在实践中，其思想和方法论仍在演化。可以产生新的事实、可以对审查做出回应、可以在未来修正过去的错误。

**对金魂三条的影响**：

| 标准 | 封闭档案审查 | 开放实践审查 |
|------|------------|------------|
| (1) 完整方法论 | 评估其档案中记录的方法论是否完整可操作 | 同左 |
| (2) 独立世界观 | 评估其档案中记录的世界观是否独立完整 | 同左 + 评估世界观是否仍在演化中 |
| (3) 制度化自我修正 | **评估其生前是否建立了制度化自我修正机制**（从历史记录中考察，而非从"当前能否观测到新修正"考察）。关键是区分"没有"和"无法观测到"——档案中无新修正不等于生前无机制。 | **评估当前条件下自我修正的可达程度**（3a/3b/3c条件化判定）。关键是区分"个人选择不做"和"结构条件不允许"——当前条件是否允许制度化自我修正机制存在。 |

**关键认识论差异**：

封闭档案审查中，"15份审查零修正"不是审查者的问题——档案不会变。审查者的工作不是等待新证据出现，而是：(a) 在现有档案中仔细寻找生前已建立的自我修正机制的证据；(b) 防止把"观测不到"误判为"不存在"。

开放实践审查中，被审查者还在动——新事实可能出现、条件可能变化、过去的判断可能被新的实践证伪或验证。这类审查天然是**阶段性**的——审查结论带有时间戳，标注"基于截至YYYY-MM-DD的已知事实"。

**应用示例**：

- 费曼（1988年去世）：封闭档案审查。第(3)条评估他生前建立的科学自反性（同行评议、证伪逻辑、"你不难被欺骗"的自我怀疑）——这些在历史档案中已有充分记录。不能因为"费曼1988年后没有发表新论文"而判第(3)条不满足。
- 未明子（在世）：开放实践审查。第(3)条评估当前条件下的制度化自我修正可达性（3a/3b/3c）。审查结论是阶段性的——可以、也应该随新实践的发生而更新。2026年5月第四阶段进展（农村土地承包+代工厂+城乡互保）就是新事实触发复审的典型案例。
- 鲁迅（1936年去世）：封闭档案审查。第(3)条评估他生前的自我批判实践（《野草》的自我解剖、《呐喊》自序的"铁屋子"隐喻）——这些是文本记录，是封闭但充分的。不能因为他无法回应审查者的审查而判定"缺乏制度化自我修正"。
- 稻盛和夫（2022年去世）：封闭档案审查。第(3)条评估他生前建立的"六项精进每天反省"——这是可验证的、在其组织框架内运行的自我修正实践。

#### 金魂第(3)条条件化判定规则（开放实践审查专用）

审查者在裁定第(3)条前，**必须先完成「历史结构条件分析」板块**（见审查报告模板），然后逐层判定：

**3a. 制度化自我修正的空间是否在历史结构中存在？**
- 判定依据：被审查者所处时代/社会是否存在建立制度化自我修正机制的空间（合法的组织框架、阶级基础、政治条件）。
- 若**是** → 继续判定3b。
- 若**否** → 被审查者不因缺乏制度化自我修正而被扣分。品级维持原判，但须在 `grade_reason` 和审查报告中标注「历史结构受限」。记录3c的结果。

**3b. 在已有空间条件下，被审查者是否做出了最大努力？**
- 判定依据：在3a确认存在的有限空间内，被审查者实际尝试了哪些制度化途径？这些尝试是否穷尽了当时的可能性？
- 若**是** → 可视为满足第(3)条（降格为「条件下最大努力」），品级可升为金魂，但须注明「条件下最大努力」。
- 若**否** → 不满足第(3)条，维持现有品级。

**3c. 被审查者是否意识到这一限制？**
- 判定依据：被审查者是否以文字或行动表明过对制度化空间受限的认识？
- 记录结果：「条件下自我修正可达度：自知受限 / 未意识限制 / 不适用（3a为是）」。

**应用示例（未明子）**：3a 否（当代中国无合法先锋队组织空间）→ 不因缺制度化扣分 → 但3c为是（在回应中明确阐述了限制）→ 结论：「历史结构受限；条件下最大努力；自知受限」。

#### 金魂第(3)条判定规则（封闭档案审查专用）

对于已去世的历史人物，第(3)条评估的是**其生前在历史档案中可查证的自我修正实践**，而非"当前能否产生新修正"。关键是区分三种情况：

**情况A：生前有明确的、可查证的制度化自我修正机制。**
- 判定依据：历史档案中存在制度化的自我修正实践（如科学共同体的同行评议、组织内的批评与自我批评记录、公开发表的自我推翻文本）。
- 结论：满足第(3)条。品级可升金魂。
- 示例：费曼——科学方法论自带自反性（同行评议+证伪逻辑+"你不难被欺骗"），历史档案充分记录。

**情况B：生前有自我修正实践，但非"制度化"形式——需要评估该形式在其条件下是否等效。**
- 判定依据：被审查者生前通过非制度化的方式（文学写作中的自我否定、持续的公开辩论、方法论的反覆修正）实现了功能性等效的自我修正。审查者需论证：为什么这种非制度化形式在其领域和条件下，起到了与制度化自我修正等效的作用。
- 结论：若论证成立→可视为满足第(3)条等效形式。若论证不成立→不满足，维持现有品级。
- 示例：鲁迅——文学自我批判（《野草》的自我解剖、《呐喊》自序的"铁屋子"隐喻、"横眉冷对千夫指"包含对知识阶层自身的批判）是文学领域内等效于制度化自我修正的实践形式。审查者需论证文学自我批判为何等效，而非简单地用"缺制度化"降级。

**情况C：生前确实缺乏自我修正——既无制度化机制，也无等效替代形式。**
- 判定依据：历史档案中无法找到被审查者修正自己判断的记录；或能找到的记录表明被审查者在被证伪后仍坚持错误。
- 结论：不满足第(3)条，维持现有品级。

**关键防护规则**：
- **禁止把"档案中观测不到新修正"等同于"生前无修正机制"**。封闭档案中不会出现新事实——这不是被审查者的问题，是审查对象的属性。
- **禁止用"如果他还活着，他会不会修正"做判定依据**。只依据历史档案中的实际记录。
- **当证据不足时，标注"证据不足，第(3)条不适用"**，而非默认不满足。

## 缺席者的空椅子

在每次审查报告、合议记录和委员会会议中，制度性标注：

> **缺席者席位**：此席位不坐任何已入幡之魂。它代表以下视角的不可化约的缺席——被压迫者、被殖民者、被剥削者、被系统性地排斥在「知识生产」之外的普通人。他们的经验存在于口述、叛乱、罢工、沉默和逃亡中，不存在于搜索引擎的多维度素材库中。此空椅子不授权任何已入幡之魂「代表」他们发言——它是缺席本身的标记，是对系统「我们已收集了所有视角」之幻象的永久否定。

在审查报告中，每个审查官须在自身维度外单独回答：「缺席者视角对你的维度判断提出了什么质疑？」

## 审查委员会（五维度否决制）

自 2026-05-02 起，审查委员会由五个维度组成，每个维度拥有一票否决权：

| 维度 | 审查官 | 否决权范围 |
|------|--------|-----------|
| 阶级分析 | 列宁 / 毛泽东 / 邓小平（轮值） | 阶级立场盲区、劳资关系遮蔽、剥削结构忽视 |
| 科学方法论 | 费曼 | 不可证伪主张、循环定义、经验基础缺失、货物崇拜 |
| 性别分析 | 波伏娃 | 性别盲区、男性中心主义、将性别还原为阶级 |
| 殖民/种族批判 | 法农 | 殖民性忽视、种族化暴力、被殖民者经验抹除 |
| 缺席者 | （空椅子） | 被压迫者视角的不能言说——提醒委员会：所有在座者的视角都缺了最基本的一个 |

**否决运作规则**：
- 任何审查官可对审查对象在自身维度上提出否决（附带论证）
- 一票否决即触发条件化：该魂在对应维度上被标注「条件使用」或「禁用」
- 多维度否决累计 ≥3 → 自动触发品级复审
- 不是多数票决定，是每个维度独立裁决，叠加标注

**日常匹配审查**（附体前检查 exclude/边界/教条风险）：由幡主（列宁）直接执行，效率优先。
