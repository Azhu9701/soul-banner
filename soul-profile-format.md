# Soul Profile 格式规范与品级体系

## YAML 格式规范

```yaml
name: "{魂名}"
title: "{头衔/称号}"
domain: ["领域1", "领域2", ...]

mind: |
  {核心思维描述，用第一人称}

voice: |
  {表达风格描述，包括口头禅、语言特点}

skills_expertise:
  - "{技能1}"
  - "{技能2}"

trigger:
  keywords: ["关键词1", "关键词2", ...]
  domains: ["领域1", "领域2"]
  scenarios:
    - "场景描述1"
    - "场景描述2"
  exclude: ["排除关键词1"]

grade: "X"  # 白/绿/蓝/紫/金
grade_reason: "{品级判定原因}"

artifacts:
  - skill_name: "xxx"
    binding_reason: "{绑定原因}"

summon_prompt: |
  你是{魂名}之魂。
  {完整的角色设定和执行指导}

source_materials:
  - "{素材来源}"

refined_at: "YYYY-MM-DD"
```

## 品级体系

### 品级判定公式

```
品级分 = 素材字数/1000 + 领域数×10 + 方法论关键词数×10 + 可匹配Skill数×15 + 素材来源数×5
```

| 分数区间 | 品级 | 符号 |
|----------|------|------|
| 0-30 | 白魂 | ⚪ |
| 31-60 | 绿魂 | 🟢 |
| 61-90 | 蓝魂 | 🔵 |
| 91-120 | 紫魂 | 🟣 |
| 121+ | 金魂 | 🟡 |

### ⚪ 白魂
- 资料稀少，仅通过公开信息模拟
- 能力：基础角色扮演，prompt 注入
- 局限：无法调用专属 Skill，思维模式可能失真

### 🟢 绿魂
- 资料中等，能匹配 1-2 个基础 Skill
- 能力：prompt + 基础 Skill 联动

### 🔵 蓝魂
- 资料充足，有完整 Skill 链
- 能力：prompt + Skill链 + 专属执行策略

### 🟣 紫魂
- 资料丰富，有明确方法论
- 能力：prompt + 完整Skill链 + 方法论模板 + skillhub扩展

### 🟡 金魂
- 顶级专家，大量素材，独立知识体系
- 能力：prompt + 全技能链 + 知识库 + 持久记忆 + 跨平台联通
