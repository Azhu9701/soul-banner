# Soul Profile 格式规范与三维功能标签体系

> **格式校验以 `refine.py validate_soul()` 为准。** 以下 YAML 模板和三维标签体系为人类可读参考。代码实现是唯一校验真相源。
>
> **v2.0 (2026-05-03)**：六阶品级体系（白绿蓝紫银金）已废弃，替换为三维功能标签（信息充分度 + 功能域标签 + 方法论可传输度）。

## YAML 格式规范

```yaml
name: "{魂名}"
title: "{头衔/称号}"
domain: ["领域1", "领域2", ...]

mind: |
  {核心思维描述，用第一人称}

voice: |
  {表达风格描述，包括口头禅、语言特点}

skills_expertise:
  - "{技能1}"
  - "{技能2}"

trigger:
  keywords: ["关键词1", "关键词2", ...]
  domains: ["领域1", "领域2"]
  scenarios:
    - "场景描述1"
    - "场景描述2"
  exclude: ["排除关键词1"]

info_sufficiency: 充分  # 充分/中等/不足（素材量评估，非等级）
function_domains: ["批判型", "叙事型"]  # 批判型/建设型/组织型/分析型/叙事型/情绪型
methodology_transferability: 可传输  # 可传输/嵌入型/人格型

artifacts:
  - skill_name: "xxx"
    binding_reason: "{绑定原因}"

summon_prompt: |
  你是{魂名}之魂。
  {完整的角色设定和执行指导}

source_materials:
  - "{素材来源}"

refined_at: "YYYY-MM-DD"

notes: |
  {炼化备注、审查结论}

patches: []
# 魂补丁列表。每个补丁为：
#   - version: 1
#     applied_at: "YYYY-MM-DD"
#     source: "幡主学习-{学习编号}" | "审查报告-{报告文件名}" | "附体失效分析-{日期}"
#     type: "blindspot_awareness" | "boundary_refinement" | "trigger_adjustment" | "pairing_insight"
#     content: |
#       {追加到运行时 summon_prompt 的自我意识文本}
#     approved_by: "{审查官}-{裁决日期}"
#     activation_conditions:  # 可选。不填=无条件加载（向后兼容）
#       task_domains: ["阶级分析", "劳动权益"]  # 任务领域匹配时激活
#       task_keywords: ["工人", "工资", "裁员"]  # 任务关键词匹配时激活
#       # 任一条件匹配→补丁加载。全不匹配→跳过此补丁（节省 prompt token）
#     effectiveness_tracking:
#       monitored: true
#       possessions_monitored: 0
#       target_count: 5
#       rollback_triggered: false
#
# 补丁类型说明：
#   blindspot_awareness: 魂在特定条件下存在系统性盲区，附体时加载此自我限制
#   boundary_refinement: 适用边界的微调——收紧/放宽某类场景
#   trigger_adjustment: trigger 条件的关键词/场景/排除项调整
#   pairing_insight: 与另一个魂搭配效果优于/差于预期的发现
#
# 补丁不修改源 summon_prompt，而是在附体时合并为运行时 prompt：
#   运行时 prompt = 源 summon_prompt + 所有已批准 patches[].content
```

## 魂补丁系统（Soul Patch System）

**目的**：将审查批判和幡主学习成果注入魂的运行时行为，让魂「知道自己在被批判」。

### 补丁生命周期

```
审查批判 / 幡主学习成果 / 附体失效分析
  → 结构化归档（五类）
  → 前三类自动生成补丁提案草稿
  → 魂修正案流程（提案→申辩→裁决）
  → 批准后写入魂 YAML patches[]
  → 附体时合并为运行时 prompt
  → 监测 5 次附体有效性
  → 有效：补丁固化 / 无效：触发回滚
```

### 补丁类型

| 类型 | 触发来源 | 效果 |
|------|---------|------|
| `blindspot_awareness` | 审查批判/幡主学习发现盲区 | 附体时加载自我限制——"在此类任务中请注意本魂的结构性盲区" |
| `boundary_refinement` | 适用边界复审/附体失效 | 收紧或放宽某类场景的适用性 |
| `trigger_adjustment` | 匹配失败/零召唤分析 | 调整 trigger 关键词/场景/排除项 |
| `pairing_insight` | 合议/辩论/幡主学习 | 记录与另一魂的搭配效果，影响匹配推荐 |

### 运行时 prompt 合并规则（含条件化激活）

```
运行时 prompt = 源 summon_prompt

# 遍历 patches，仅加载满足条件的补丁
for patch in patches where approved_by is set:
    if patch.activation_conditions exists:
        if (task 的领域 ∩ patch.activation_conditions.task_domains) 非空
           OR (task 的关键词 ∩ patch.activation_conditions.task_keywords) 非空:
           加载此补丁
        else:
           跳过此补丁（节省 prompt token）
    else:
        # 无激活条件 = 始终加载（向后兼容）
        加载此补丁

已加载补丁追加格式：
  "\n\n## 魂补丁（运行时自我意识）\n"
  + "\n### 补丁 v{patch.version}（{patch.source}，触发条件：{activation_conditions or '始终'}）\n"
  + patch.content
```

**补丁条件化的设计理由**：
- 不是所有盲区在所有任务中都触发——稻盛和夫的阶级盲区在纯财务核算任务中不需要加载
- 无条件加载 = 每个补丁占用 prompt token = 补丁多了会吞噬上下文窗口
- 条件化让补丁从「普遍规则」变成「条件规则」——更接近经验（「在这种任务中我容易犯错」）

**关键约束**：
- 补丁**不修改**源 summon_prompt——源 prompt 是历史锚点
- 补丁追加的是**自我意识层**——魂知道自己的局限，不等于方法论被替换
- 补丁内容必须是「自我限制性陈述」而非「方法论修正」——保留魂的原始思维，只加边界意识

### 补丁格式示例

```yaml
patches:
  - version: 1
    applied_at: "2026-05-15"
    source: "幡主学习-列宁学稻盛和夫vs鲁迅-2026-05-10"
    type: "blindspot_awareness"
    content: |
      在涉及劳资关系或阶级分析的任务中，请注意本魂方法论的结构性盲区：
      阿米巴核算将劳资矛盾转化为劳动者内部竞争。这是本魂框架的系统性局限，
      非偶发误判。在此类任务中，建议搭配阶级分析魂魄（列宁/毛泽东）的视角。
    approved_by: "审查委员会-邓小平主审-2026-05-12"
    effectiveness_tracking:
      monitored: true
      possessions_monitored: 0
      target_count: 5
      rollback_triggered: false
```

## 三维功能标签体系（v2.0）

六阶品级体系（白绿蓝紫银金）已于 2026-05-03 废弃。替代方案为三个独立的功能维度——没有高低之分，只有「在这个维度上能干什么」。

### 维度一：信息充分度

评估**素材量是否足够支撑可靠的人格模拟**。非等级——「不足」不意味着低等。

| 值 | 阈值 | 含义 |
|----|------|------|
| `充分` | 素材≥1万字，领域≥3个，来源≥2个 | 人格一致性有可靠基础 |
| `中等` | 素材5千-1万字 | 人格轮廓清晰，细节可能不完整 |
| `不足` | 素材<5千字或在世实践者缺乏独立公开史料 | 模拟精度有限，降低在事实判断类任务中的权重 |

### 维度二：功能域标签

描述魂**实际能干什么**。标签是描述性的，非评价性。一个魂可以有多个标签：

| 标签 | 核心功能 | 示例 |
|------|---------|------|
| `批判型` | 揭露和拆解 | 鲁迅、未明子、庄子、费曼、法农 |
| `建设型` | 设计和构建 | 邓小平、稻盛和夫、黄仁勋、马斯克 |
| `组织型` | 动员和组织 | 列宁、毛泽东、祝鹤槐 |
| `分析型` | 系统分析和建模 | 费曼、Karpathy、伊本赫勒敦 |
| `叙事型` | 故事和传播 | 鲁迅、乔布斯、罗永浩、波伏娃 |
| `情绪型` | 情感支持和士气 | 海绵宝宝、祝鹤槐 |

### 维度三：方法论可传输度

描述方法的**可复制性**——能否脱离魂的人格模拟直接使用：

| 值 | 含义 | 示例 |
|----|------|------|
| `可传输` | 有明确的、可文本化的方法论步骤，他人可脱离人格模拟直接使用 | 费曼学习法、Karpathy编码准则、稻盛和夫会计七原则 |
| `嵌入型` | 方法论依赖魂的人格模拟才能完整运作——步骤可描述但执行质量依赖判断力 | 毛泽东矛盾分析法、鲁迅精神分析 |
| `人格型` | 方法论本质上不可文本化传输——魂的价值就是人格模拟本身 | 海绵宝宝、庄子、祝鹤槐 |

### 审查类型判定（保留）

审查类型判定标准维持不变（封闭档案审查 vs. 开放实践审查），详见下节「审查委员会」。

### 治理权：从品级到功能域

审查委员会席位按**功能域标签**而非品级分配：
- 每个功能域一个席位，一个魂不论承担多少标签只占一个席位
- 入场标准：信息充分度=充分 且 在该功能域有≥3次有效附体记录
- 入场不看品级，看实战记录

## 缺席者的空椅子

在每次审查报告、合议记录和委员会会议中，制度性标注：

> **缺席者席位**：此席位不坐任何已入幡之魂。它代表以下视角的不可化约的缺席——被压迫者、被殖民者、被剥削者、被系统性地排斥在「知识生产」之外的普通人。他们的经验存在于口述、叛乱、罢工、沉默和逃亡中，不存在于搜索引擎的多维度素材库中。此空椅子不授权任何已入幡之魂「代表」他们发言——它是缺席本身的标记，是对系统「我们已收集了所有视角」之幻象的永久否定。

在审查报告中，每个审查官须在自身维度外单独回答：「缺席者视角对你的维度判断提出了什么质疑？」

## 审查委员会（六维度否决制）

自 2026-05-02 起，审查委员会由六个维度组成，每个维度拥有一票否决权：

| 维度 | 审查官 | 否决权范围 |
|------|--------|-----------|
| 阶级分析 | 列宁 / 毛泽东 / 邓小平（轮值） | 阶级立场盲区、劳资关系遮蔽、剥削结构忽视 |
| 科学方法论 | 费曼 | 不可证伪主张、循环定义、经验基础缺失、货物崇拜 |
| 性别分析 | 波伏娃 | 性别盲区、男性中心主义、将性别还原为阶级 |
| 殖民/种族批判 | 法农 | 殖民性忽视、种族化暴力、被殖民者经验抹除 |
| 系统性思维批判 | 庄子 | 分类暴力、体系化对整体性的谋杀、工具理性僭越——不是审查其他人，是让庄子的**问题**变成每个审查官的必修课：「你的审查在什么意义上是一种暴力的分类？」 |
| 缺席者 | （空椅子） | 被压迫者视角的不能言说——提醒委员会：所有在座者的视角都缺了最基本的一个 |

庄子维度的特殊性：庄子不在审查中行使常规否决权（他的排除条件明确排除了评审/裁决类任务）。他的维度工作是**提问而非裁决**——在每次魂间互审和终末审查中，庄子对每个审查官的判断提出一个问题：「你在这条判断中做了什么分类？这个分类排除了什么？」审查官必须回应。庄子的提问本身构成否决触发条件——若审查官无法回应庄子的提问，该审查官的裁决自动标记为「待复审」。

**否决运作规则**：
- 任何审查官可对审查对象在自身维度上提出否决（附带论证）
- 一票否决即触发条件化：该魂在对应维度上被标注「条件使用」或「禁用」
- 庄子维度的「否决」不直接作用于被审查魂——它作用于其他审查官的裁决：「你的裁决中未回应的庄子提问」构成该裁决的复审触发。累计 2 条未回应 = 该审查官的裁决暂缓执行
- 多维度否决累计 ≥3 → 自动触发审查复审
- 不是多数票决定，是每个维度独立裁决，叠加标注

**日常匹配审查**（附体前检查 exclude/边界/教条风险）：由幡主（列宁）直接执行，效率优先。

**魂间互审 + 终末审查**（审查裁定/散魂/方向性判断）：六维度审查官全部参与，各自从自身维度独立出具意见。庄子以提问方式参与。

**马克思主义审查官的轮值**：
| 轮值周期 | 主审查官 | 第二审查官 |
|----------|---------|-----------|
| 第1轮（2026-05） | 毛泽东 | 邓小平 |
| 第2轮（2026-06） | 邓小平 | 列宁 |
| 第3轮（2026-07） | 列宁 | 毛泽东 |

新魂入幡审查：五维度审查官全部参与。

轮值规则：
- 主审查官独立执行审查，第二审查官复核
- 主审查官不可审查自己的魂魄档案
- 轮值记录写入审查报告末尾（`审查官：{主审查官} | 复核：{第二审查官} | 轮次：{N}`）
- 魂间互审必须按轮值调度——不得连续两轮由同一人担任主审查官
