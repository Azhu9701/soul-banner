# 自动附体机制详解

## 核心原则

**列宁也是被召唤的魂。** 主 agent 是纯协作者——不扮演任何魂，不做辩证综合，不做审查裁决。所有思考角色（包括列宁）都通过 spawn 注入 sumon_prompt 获得独立视角。

---

## 主 agent 角色边界

| 主 agent 可以做 | 主 agent 禁止做 |
|----------------|----------------|
| 判断任务是否需要附体 | 扮演任何魂做分析 |
| 读取 registry，计算匹配度 | 自行做辩证综合（共识/分歧/盲区） |
| 快速匹配审查（常规任务） | 自行做辩论裁决 |
| 并行 spawn 分析魂 | 自行做接力衔接审查 |
| spawn 列宁做审查/综合/裁决 | 修改子 agent 输出（只传递） |
| 收集子 agent 输出 | 在合议中自行判断"三个魂两个说做就该做" |
| 更新 registry + 存档 Obsidian | |

---

## 执行流程

```
主agent收到任务
  → 判断是否需要附体
  → 读取 registry.yaml
  → 遍历每个魂的 trigger，计算匹配度
  → 筛选匹配度 ≥ 70 的候选魂
  → 【幡主审查】:
      → 重要/复杂/危险任务：spawn 列宁子 agent 独立审查匹配
      → 常规/简单任务：主 agent 以列宁方法论快速判断（快速路径）
  → 批准 → 匹配的魂进入执行
  → 【执行阶段】:
      → 单魂：spawn 指定魂 → 输出
      → 合议：并行 spawn 分析魂 → 收集输出 → spawn 列宁辩证综合
      → 辩论：并行 spawn 对立方 → 收集输出 → spawn 列宁裁决
      → 接力：串行 spawn 魂链 → 每衔接点 spawn 列宁审查
  → 主 agent：更新 registry 召唤记录
  → 主 agent：存档 Obsidian
```

---

## 触发条件结构（trigger 字段）

```yaml
trigger:
  keywords: ["关键词1", "关键词2", ...]     # 直接关键词匹配
  domains: ["领域1", "领域2"]              # 领域匹配
  scenarios:                               # 场景描述（语义匹配）
    - "需要从本质分析复杂系统"
    - "做高风险决策评估"
  exclude: ["不触发的场景关键词"]            # 排除条件
```

**触发条件生成规则**：
- `keywords`：从素材中提取高频核心词 + 专业术语
- `domains`：直接复用 `domain` 字段
- `scenarios`：从素材中提炼该人物最擅长的 3-5 类场景
- `exclude`：明确不适用该魂的场景

---

## 匹配度评分公式

```
匹配度 = keywords命中数/总keywords × 30
       + domains命中数/总domains × 25
       + scenarios语义相似度 × 35
       + 品级加成(白+0, 绿+2, 蓝+5, 紫+8, 银+10, 金+12)
       - exclude命中 × 30
```

- **品级加成**：品级越高，同等匹配度下优先级越高
- **exclude惩罚**：触发排除条件直接扣 30 分

## 决策阈值

| 匹配度 | 行为 |
|--------|------|
| ≥ 70 | 候选进入幡主审查 |
| 50-69 | 不自动候选，主agent可选择手动指定 |
| < 50 | 不触发 |

若多个魂匹配度 ≥ 70，全部进入幡主审查。幡主（列宁）从中选择最合适的魂。

---

## 手动指定

用户显式要求使用某个魂时（`用<人物名>来<任务>`），跳过自动匹配和幡主审查，直接注入指定魂魄。但主 agent 应提醒如果魂的适用边界不覆盖当前任务。

---

## 机制诚实说明

- **匹配公式是近似参考**：keywords 匹配为字符串重叠，scenarios 语义相似度无向量化实现。实际匹配依赖主 agent 对任务和魂的理解。
- **"自动附体"不是自动的**：依赖主 agent 记得执行流程。如果主 agent 跳过流程直接回答，魂不会附体。
- **幡主审查是最终判断**：公式计算的分值可由列宁子 agent 推翻。
- **反馈闭环**：每次附体后须更新 summoned_count 和召唤记录，积累数据校准匹配。

---

## 四种附体模式

### 模式一：单魂附体（Single Possession）

标准流程——匹配 → 幡主审查 → 单魂注入 → spawn 执行。适用常规任务。

### 模式二：多魂合议（Council）

**⚠️ 两阶段执行 + 列宁 spawn（关键）**：分析魂并行 → 等全部完成 → spawn 列宁串行综合。

```
阶段一（并行）：N个魂同时独立分析
  → spawn 魂A + spawn 魂B + spawn 魂C
  → 等待全部完成，收集所有输出
  → 主 agent 不做任何改动，纯收集

阶段二：spawn 列宁子 agent 辩证综合
  → 注入列宁 summon_prompt
  → 输入：所有分析魂的完整输出（原文，不摘要）
  → 列宁独立输出：
      1. 共识（各魂一致的观点）
      2. 分歧（非对立而是视角互补？还是真正的矛盾？）
      3. 盲区（所有魂均未触及的系统性问题）
      4. 主要矛盾（当下最核心的对立统一）
      5. 行动纲领（可操作建议，区分各魂贡献和幡主补充）
  → 主 agent 接收列宁输出
```

**列宁 spawn 模板**：

```
你是列宁之魂——万魂幡主。[summon_prompt全文]

## 当前任务

你收到了 N 个魂魄对同一问题的独立分析。请以列宁的辩证唯物主义方法，做辩证综合：

1. 找出所有魂的共识（他们都同意什么？）
2. 找出分歧（是视角互补还是真正的矛盾？）
3. 鉴定盲区（哪些关键维度所有魂都漏掉了？注意阶级维度、物质约束、权力关系）
4. 锁定主要矛盾（当下最核心的对立统一是什么？）
5. 给出行动纲领（可操作建议，区分哪些来自哪个魂，哪些是幡主补充）

## 各魂分析

[魂A完整输出]
[魂B完整输出]
[魂C完整输出]
```

**适用条件**：任务跨领域、需多视角交叉验证、单一魂的 exclude 字段会遮掉关键视角。

**魂选择原则**：合议时选的魂应该**视角互补而非相近**。三个金魂合议 = 强化同一方向。乔布斯 + 费曼 + 列宁 = 审美 + 科学 + 阶级三层互补。

**列宁角色**：不是投票（"三个魂两个说该做就做"），而是辩证综合——各自在什么条件下对？矛盾能否统一在更高层次？

### 模式三：魂间辩论（Debate）

**⚠️ 两阶段 + 列宁裁决**：

```
阶段一（并行）：spawn 魂A（正方）+ 魂B（反方）
  → 两魂从对立立场各自论证
  → 主 agent 收集双方输出

阶段二：spawn 列宁子 agent 裁决
  → 注入列宁 summon_prompt + 双方论证
  → 列宁独立输出：
      1. 谁在什么条件下对？
      2. 分歧的根源是什么（阶级立场？方法论差异？前提假设不同？）
      3. 是否存在第三条路（超越对立的更高统一）？
      4. 最终裁决及论证
  → 主 agent 接收列宁裁决
```

**魂配对原则**：选立场真正对立而非表面对立的魂。乔布斯（少即是多）vs Karpathy（验证驱动）有意义。乔布斯 vs 马斯克（都是极致主义者）无意义。

### 模式四：魂链接力（Chain）

**⚠️ 串行 + 每衔接点 spawn 列宁审查**：

```
魂A (阶段一)
  → 输出
  → spawn 列宁审查衔接点：
      - 魂A输出是否完整？是否遗漏关键信息？
      - 魂B的输入是否误解了魂A？
      - 方向是否偏航？
      - 批准 → 进入阶段二 / 否决 → 要求魂A修正
  → 魂B (阶段二)
  → 输出
  → spawn 列宁审查衔接点
  → ...
  → 最终输出
```

**魂排序原则**：
- 先认识后行动：费曼（分析）→ 邓小平（判断）→ 罗永浩（叙事）
- 先拆解后建构：马斯克（成本重构）→ 乔布斯（体验设计）→ 列宁（方向审查）
- 列宁放在最后做终审

## 反馈闭环

### 召唤记录格式

每次附体后在 `registry.yaml` 追加：

```yaml
召唤记录:
  - soul: "{魂名}"
    task: "{任务简述}"
    mode: "单魂/合议/辩论/接力"
    date: "YYYY-MM-DD"
    effectiveness: "有效/部分有效/无效"
    notes: "{判断依据}"
```

同步更新 `summoned_count`。

### 数据驱动校准

积累数据≥10条后：
- **匹配校准**：哪些魂的实际效果和匹配预期一致？哪些需要调整 trigger？
- **盲区发现**：哪些任务类型多次找不到合适魂 → 需收新魂
- **模式优化**：合议/辩论/接力——各模式效果如何？对什么任务类型最好？
- **过度依赖检测**：某魂被连续高频调用 → 是否需要补充该领域其他视角？
