## v2026.05.02-5

### 未明子七条改造建议执行 — 反消费主义改造

基于未明子对万魂幡的意识形态分析（四维坐标定位 + 教条化风险诊断），执行全部七条改造建议：

**1. 自我否定环节强制化**
- 每次附体后强制问「哪个预设被动摇了？」，无法回答 → 标记「消费性使用」
- 连续 3 次消费性使用 → 强制进入幡主学习模式
- CLAUDE.md、SKILL.md 新增相关章节

**2. 庄子第六维度否决权**
- 审查委员会从五维度扩展为六维度：新增「系统性思维批判」由庄子主持
- 庄子以提问而非裁决的方式运作——审查官无法回应庄子提问 → 裁决自动标记「待复审」
- 累计 2 条未回应 → 该审查官的裁决暂缓执行
- 修改：soul-profile-format.md、SKILL.md、README.md、README_zh.md

**3. 匹配从品级优先改认知距离优先**
- 移除金魂品级加成（score *= 1.05）
- 新增 `apply_cognitive_distance()`：使用越频繁的魂轻微降权（-0.05），零召唤魂 +0.05
- 公式：freshness = 1.0 / (1 + ln(usage + 1))，归一化到 [-0.05, +0.05]
- 修改：scripts/match.py

**4. 附体前使用者预设声明**
- 匹配前主 agent 必须追问三问：已有判断/担忧/未知
- 预设不用于筛选魂——用于附体结束后的自我否定对照
- 修改：CLAUDE.md

**5. 使用者参与环节**
- 辩证综合后追问两问：「最没想到的部分」「早就知道的部分」
- 第 1 问标记真正的学习，第 2 问暴露消费行为
- 修改：CLAUDE.md

**6. 魂生成链路透明化**
- Spawn 模板末尾附带生成链路说明：「本魂基于 XX 日期收魂素材炼化……是 AI 模拟，不是本人」
- 修改：CLAUDE.md

**7. 空椅子拷问环节**
- 每次附体报告末尾追加：「在这次分析中，谁的利益被代表？谁的发言权没有被给？」
- 回答追加到 Obsidian 存档报告最后一节
- 修改：CLAUDE.md、SKILL.md

## v2026.05.02-2

### 性能优化 — 匹配审查 Token 消耗 -72%

- **幡主审查零文件读取**：新增硬性禁读规则，审查仅基于 prompt 注入的预筛选结果 + handbook 做判断（8 tool uses → 0）
- **match.py `--no-gold-review`**：省略 gold_review 字段，预筛选输出节省 ~1,500 tokens
- **新增幡主审查专用 spawn 模板**（CLAUDE.md:90-102），prompt 中强制包含「禁止读取任何文件」
- 实测：40.4k tokens → 11.2k tokens（-72%），3m13s → 58s（-70%）

### 免费工具审计与改造

- **全依赖审计**：所有核心功能（收魂、炼化、审查、附体、合议）确认零额外费用可运行
- **搜索回退链重排**：WebSearch + WebFetch（免费内置，零配置）优先于 byted-web-search（需火山引擎账号）
- **byted-web-search 降级为可选付费项**，移除出主回退链
- **SKILL.md 新增「免费使用指南」章节**，含穷哥们快速上手指令
- **外部 Skill 全部标注费用状态**（markitdown 免费开源 / humanizer 纯 LLM / graphify 可选免费 等）

### 匹配脚本增强

- **match.py exclude 两级判定**（hard/soft）：长词精确命中=硬排除（×0.1），模糊/附属=软排除（×0.5），降低排除误伤率
- **排除清单结构化**：区分硬排除预警和软排除提示

### 事务脚本修复

- **transact.py obsidian_content**：改为传路径而非主 agent 读内容，避免大文件 token 浪费
